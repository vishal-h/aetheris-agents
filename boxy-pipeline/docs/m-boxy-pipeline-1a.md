# Milestone: boxy-pipeline-1a — Data Layer

**Goal.** Build the foundational data layer that everything in boxy-pipeline
builds on: a clean, versioned catalog extracted once from the Boxy Excel,
a structured JSON representation of completed Sales Orders, and a refactored
resolver that reads from the extracted catalog rather than parsing the Excel
on every run.

**What exists before this milestone.**
- `scripts/plan_extractor.py` — extracts 20-20 plan codes → `PlanComponent` JSON
- `scripts/catalog_resolver.py` — resolves plan codes against Boxy Excel (parses
  Excel on every run; fragile against format changes)
- `scripts/order_formatter.py` — writes Boxy Order Form xlsx
- `main.py` — end-to-end CLI
- `scripts/schema.py` — `PlanComponent`, `CatalogItem`, `ResolvedItem`,
  `PipelineResult`

**What exists after.**
- `scripts/catalog_extractor.py` — extracts all 5 series from Boxy Excel →
  `data/catalog.jsonl` (versioned, timestamped, gitignored)
- `scripts/so_extractor.py` — extracts a Boxy Sales Order PDF →
  `data/projects/{name}/sales_order.json`
- `scripts/schema.py` — extended with `CatalogEntry`, `SOHeader`, `SOLineItem`,
  `SalesOrder` dataclasses
- `scripts/catalog_resolver.py` — refactored to read `catalog.jsonl` instead of
  parsing Excel; Excel parsing removed from the hot path
- `data/catalog.jsonl` — on disk locally; not committed

**Out of scope.**
- 20-20 → Boxy code mapping table (planned for milestone 2)
- Catalog annotation/enrichment UI
- Reconciliation between plan and SO (planned for milestone 2)
- Any change to `order_formatter.py`, `main.py`, or `plan_extractor.py`

**Sample files (gitignored, must be present at `data/samples/`).**
- `Updated_Boxy_MSRP_Sales_Order_Form.xlsx` — Boxy MSRP catalog (all series)
- `SO86708_Aria_Joey.pdf` — completed Boxy Sales Order (ground truth for t2)

## Canonical schema additions (`schema.py`)

New dataclasses added in t1. Existing dataclasses (`PlanComponent`,
`CatalogItem`, `ResolvedItem`, `PipelineResult`) are **not modified**.

```python
@dataclass
class CatalogEntry:
    """One item from the extracted catalog — all series, all colors."""
    sku: str                    # e.g. "W2739-2001"
    base_code: str              # e.g. "W2739" (leading-digit stripped)
    raw_code: str               # as it appears in the Excel, e.g. "W2739"
    series: str                 # "1000" … "5000"
    color_code: str             # "2001", "2004", etc.
    color_name: str             # "Ivory White", "Mingo Oak", etc.
    description: str
    cabinet_type: str           # first segment before comma
    width_in: Optional[float]
    height_in: Optional[float]
    depth_in: Optional[float]
    msrp: float
    # Enrichment fields — populated manually or by future tooling
    mapped_20_20_codes: list[str]   # known 20-20 equivalents, e.g. ["BLB42FHL"]
    notes: Optional[str]            # free-text annotation
    # Provenance
    catalog_version: str        # ISO date of extraction, e.g. "2026-06-14"
    source_file: str            # filename of the source Excel


@dataclass
class SOLineItem:
    """One line item from a Boxy Sales Order PDF."""
    line: int
    sku: str                    # e.g. "3DB30-2004"
    description: str
    qty: int
    unit_price: float
    amount: float
    special_request: Optional[str]
    is_fee: bool                # True for Assembly Fee, Delivery fee, etc.
    is_accessory: bool          # True for Accs-* items


@dataclass
class SOHeader:
    order_number: str           # e.g. "SO86708"
    order_date: str             # ISO date
    customer: str
    bill_to: str
    ship_to: str
    estimate_number: Optional[str]
    payment_term: Optional[str]
    po_number: Optional[str]
    account_manager: Optional[str]
    ship_date: Optional[str]
    memo: Optional[str]


@dataclass
class SalesOrder:
    header: SOHeader
    line_items: list[SOLineItem]
    subtotal: float
    tax_total: float
    total: float
    source_file: str
    extracted_at: str           # ISO datetime
```

## `catalog.jsonl` format

One JSON object per line. Each line is a `CatalogEntry` dict. The file is
versioned by its `catalog_version` field (ISO date of extraction) and by the
filename: `catalog-{YYYY-MM-DD}.jsonl`, symlinked or referenced as
`catalog.jsonl`.

```jsonl
{"sku": "W2739-2001", "base_code": "W2739", "raw_code": "W2739", "series": "2000", "color_code": "2001", "color_name": "Ivory White", "description": "...", "cabinet_type": "Wall Cabinet", "width_in": 27.0, "height_in": 39.0, "depth_in": 12.0, "msrp": 450.8, "mapped_20_20_codes": [], "notes": null, "catalog_version": "2026-06-14", "source_file": "Updated_Boxy_MSRP_Sales_Order_Form.xlsx"}
{"sku": "W2739-2004", "base_code": "W2739", ...}
```

Total expected lines: all SKU × color combinations across all 5 series.

**Scope.** `scripts/catalog_extractor.py` reads the Boxy MSRP Excel (all 5
Price List sheets) and writes `data/catalog.jsonl` — one `CatalogEntry` per
line. Also extends `scripts/schema.py` with the new dataclasses

- `docs/m-boxy-pipeline-1a.md §Repository layout changes` — file locations.

**Approach.**
The Excel parsing logic already exists in `catalog_resolver.py`
(`load_catalog` function). The extractor reuses that logic but:
- Extracts all 5 series (not just 2000)
- Emits `CatalogEntry` dicts (not `CatalogItem` — note the distinction:
  `CatalogItem` is the existing resolver type; `CatalogEntry` is the new
  master catalog type with enrichment fields)
- Sets `mapped_20_20_codes: []` and `notes: null` on all entries (enrichment
  fields are empty at extraction time)
- Sets `catalog_version` to today's ISO date (`datetime.date.today().isoformat()`)
- Sets `source_file` to the basename of the input Excel
- Writes output as JSONL to `data/catalog.jsonl` (one dict per line,
  no pretty-printing)
- Also prints a summary to stdout: total entries, entries per series

The extractor is a **one-shot script**, not part of the pipeline hot path.
It is run manually when the Boxy catalog is updated.

### t2 — SO extractor

**Scope.** `scripts/so_extractor.py` reads a Boxy Sales Order PDF
(like `SO86708_Aria_Joey.pdf`) and writes a `SalesOrder` JSON file to
`data/projects/{project_name}/sales_order.json`.

**Contract refs.**
- `scripts/schema.py` — `SalesOrder`, `SOHeader`, `SOLineItem` are the output
  types (added in t1); read the file, do not restate fields.
- `docs/m-boxy-pipeline-1a.md §t2 — SO extractor` — this section.
- `docs/m-boxy-pipeline-1a.md §sales_order.json format` — output format.

**Approach.**
The SO PDF (`SO86708_Aria_Joey.pdf`) has embedded text — pdfplumber extracts
it cleanly. The document structure:

- **Page 1:** header block (Bill To, Ship To, Customer, order metadata),
  then a table starting with headers `Quantity | Item | Special Request |
  Rate | Amount`
- **Pages 1–3:** line items (one per row; Item column has bold SKU on first
  line, description on second)
- **Page 3:** totals block (`Subtotal`, `Tax Total`, `Total`)
- **Page 4:** terms and conditions — skip entirely

Parsing strategy:
1. Extract text with `pdfplumber`. Use `extract_table()` on each page to
   capture the line item table rows.
2. Header fields: use regex on the first page text to extract order number
   (`#SO\d+`), date, customer, bill-to, ship-to, payment term, PO#,
   account manager, ship date, memo.
3. Line items: rows where col 0 (Quantity) is a digit. The Item column
   contains `{SKU}\n{description}` — split on first newline.
4. Fee rows: rows where col 0 is a digit but the item text matches known
   fee patterns (`Assembly Fee`, `Modify-S`, `Modify-L`, `Delivery fee`,
   `Installation fee`). Set `is_fee: True`.
5. Accessory rows: SKU starts with `Accs-`. Set `is_accessory: True`.
6. Totals: parse `Subtotal`, `Tax Total`, `Total` from last page text.

Known edge cases from `SO86708_Aria_Joey.pdf`:
- `Box-SB36` — no description line (special case item)
- `Box-WP2493` — modified items with description in Special Request
- Multi-quantity rows (e.g. qty=2) — parse as single line item with qty field
- Amount column may show `$588.35` or `$1,014.70` — strip `$` and `,`

**Touches.**
- `scripts/so_extractor.py` (new)
- `tests/test_so_extractor.py` (new)

**Do not generate.**
- Any modification to `schema.py` (types added in t1)
- Changes to any other script

**Done-check.**

### t3 — Resolver refactor

**Scope.** Refactor `scripts/catalog_resolver.py` to read from
`data/catalog.jsonl` (produced by t1) instead of parsing the Boxy Excel on
every run. The resolver's public API (`resolve()`, `parse_finish()`) and
output types (`ResolvedItem`) are unchanged — `main.py` and
`test_catalog_resolver.py` must continue to pass without modification.

**Contract refs.**
- `scripts/schema.py` — `CatalogEntry` (new, from t1) replaces the internal
  Excel-parsing path; `ResolvedItem` output type is unchanged.
- `docs/m-boxy-pipeline-1a.md §t3 — Resolver refactor` (this section).
- `docs/m-boxy-pipeline-1a.md §catalog.jsonl format` — input format.

**Approach.**
Currently `catalog_resolver.py` calls `load_catalog(path: Path)` which
opens the Excel, parses all Price List sheets, and builds an in-memory index.
This function is called on every pipeline run.

Replace `load_catalog` with `load_catalog_jsonl(path: Path)` which reads
`catalog.jsonl` and builds the same index structure
(`dict[str, list[CatalogItem]]`) from `CatalogEntry` dicts. The index
structure is identical — the change is only in how it's populated.

Keep `load_catalog` (Excel path) as a deprecated fallback, clearly marked.
The `--catalog` flag on the CLI continues to accept either an xlsx or a jsonl
file, detected by file extension.

The `CatalogEntry` → `CatalogItem` mapping:
```python
CatalogItem(
    sku=entry['sku'],
    series=entry['series'],
    color_code=entry['color_code'],
    color_name=entry['color_name'],
    description=entry['description'],
    cabinet_type=entry['cabinet_type'],
    width_in=entry['width_in'],
    height_in=entry['height_in'],
    depth_in=entry['depth_in'],
    msrp=entry['msrp'],
)
```

**Performance expectation.** Loading from JSONL should be measurably faster
than Excel parsing. Include a timing comparison in the implementation notes
(not in the done-check, just noted).
