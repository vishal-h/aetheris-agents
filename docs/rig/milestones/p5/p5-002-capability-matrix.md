# rig/p5: Capability matrix view

## Context

The capability matrix (`docs/capability-matrix.md`) is the canonical inventory
of every agent and script in the repo. This issue surfaces it as a browsable,
launchable catalogue inside Rig — a new section under the Harness sidebar
entry. Agents can be launched via the Orchestrator flow. Scripts are readonly
reference material.

All work is in `aetheris-agents/rig/`.

## What to build

### New Tauri command: `capability_matrix_load`

Add `src-tauri/src/commands/capability_matrix.rs`.

Reads `docs/capability-matrix.md` from `AETHERIS_AGENTS_PATH` and parses it
into structured data. No new state struct — derives path from env var at call
time.

```rust
#[derive(serde::Serialize, Clone)]
pub struct MatrixAgent {
    pub file:        String,    // e.g. "payslip_orchestrator.exs"
    pub label:       String,    // e.g. "Payslip Orchestrator"
    pub tools:       Vec<String>,
}

#[derive(serde::Serialize, Clone)]
pub struct MatrixScript {
    pub file:        String,    // e.g. "payslip_compute.py"
    pub purpose:     String,    // one-line description
}

#[derive(serde::Serialize, Clone)]
pub struct MatrixUseCase {
    pub title:       String,    // e.g. "Payslip"
    pub agents:      Vec<MatrixAgent>,
    pub scripts:     Vec<MatrixScript>,
}

#[derive(serde::Serialize)]
pub struct CapabilityMatrix {
    pub use_cases:   Vec<MatrixUseCase>,
    pub generated_at: Option<String>,  // from the "_Generated by_" line if present
}

#[tauri::command]
pub fn capability_matrix_load() -> Result<CapabilityMatrix, String> {
    let agents_path = std::env::var("AETHERIS_AGENTS_PATH")
        .map_err(|_| "AETHERIS_AGENTS_PATH not set".to_string())?;

    let path = std::path::Path::new(&agents_path)
        .join("docs")
        .join("capability-matrix.md");

    let raw = std::fs::read_to_string(&path)
        .map_err(|e| format!("read failed: {}", e))?;

    parse_matrix(&raw)
}
```

**Parser implementation:**

The Markdown structure is regular enough to parse line-by-line:

```rust
fn parse_matrix(raw: &str) -> Result<CapabilityMatrix, String> {
    let mut use_cases: Vec<MatrixUseCase> = vec![];
    let mut current_uc: Option<MatrixUseCase> = None;
    let mut in_agents_table  = false;
    let mut in_scripts_table = false;
    let mut generated_at: Option<String> = None;

    for line in raw.lines() {
        let line = line.trim();

        // "_Generated by `agents/capability_matrix.exs`…"
        if line.starts_with("_Generated") {
            generated_at = Some(line.trim_matches('_').to_string());
            continue;
        }

        // "## Payslip" — new use case
        if line.starts_with("## ") {
            if let Some(uc) = current_uc.take() {
                use_cases.push(uc);
            }
            current_uc = Some(MatrixUseCase {
                title:   line.trim_start_matches("## ").to_string(),
                agents:  vec![],
                scripts: vec![],
            });
            in_agents_table  = false;
            in_scripts_table = false;
            continue;
        }

        // "### Agents" / "### Scripts"
        if line.starts_with("### Agents") {
            in_agents_table  = true;
            in_scripts_table = false;
            continue;
        }
        if line.starts_with("### Scripts") {
            in_scripts_table = true;
            in_agents_table  = false;
            continue;
        }

        // Skip non-table lines and headers
        if !line.starts_with('|') { continue; }

        // Skip table header separator "| --- |"
        if line.contains("---") { continue; }

        // Skip column header row (contains "Agent file" or "Script")
        if line.contains("Agent file") || line.contains("Script") { continue; }

        let cols: Vec<&str> = line
            .split('|')
            .map(|c| c.trim())
            .filter(|c| !c.is_empty())
            .collect();

        if let Some(ref mut uc) = current_uc {
            if in_agents_table && cols.len() >= 3 {
                // | file | label | tools |
                let tools: Vec<String> = cols[2]
                    .split(',')
                    .map(|t| t.trim().trim_matches('`').to_string())
                    .filter(|t| !t.is_empty())
                    .collect();
                uc.agents.push(MatrixAgent {
                    file:  cols[0].to_string(),
                    label: cols[1].to_string(),
                    tools,
                });
            } else if in_scripts_table && cols.len() >= 2 {
                // | file | purpose |
                uc.scripts.push(MatrixScript {
                    file:    cols[0].to_string(),
                    purpose: cols[1].to_string(),
                });
            }
        }
    }

    // Push last use case
    if let Some(uc) = current_uc {
        use_cases.push(uc);
    }

    // Drop the Summary and Overlap Report sections (they have ## headers
    // but no agents/scripts tables — they'll be empty use cases)
    let use_cases = use_cases
        .into_iter()
        .filter(|uc| !uc.agents.is_empty() || !uc.scripts.is_empty())
        .collect();

    Ok(CapabilityMatrix { use_cases, generated_at })
}
```

### `src/hooks/useCapabilityMatrix.ts`

```typescript
import { useState, useEffect } from 'react';
import { invoke } from '@tauri-apps/api/core';
import { CapabilityMatrix } from './types';

export function useCapabilityMatrix() {
  const [matrix,  setMatrix]  = useState<CapabilityMatrix | null>(null);
  const [loading, setLoading] = useState(false);
  const [error,   setError]   = useState<string | null>(null);

  useEffect(() => {
    setLoading(true);
    invoke<CapabilityMatrix>('capability_matrix_load')
      .then((m) => { setMatrix(m);  setLoading(false); })
      .catch((e) => { setError(String(e)); setLoading(false); });
  }, []);

  return { matrix, loading, error };
}
```

function UseCaseSection({
  useCase,
  onLaunch,
}: {
  useCase:  MatrixUseCase;
  onLaunch: (agent: MatrixAgent) => void;
}) {
  const [open, setOpen] = useState(true);

  return (
    <div className="border rounded-md mb-3">
      {/* Header */}
      <button
        className="w-full flex items-center gap-2 px-4 py-3 text-left

            <div className={useCase.agents.length > 0 ? 'border-t' : ''}>
              <p className="px-4 py-1.5 text-xs font-semibold text-muted-foreground
                            uppercase tracking-wide bg-muted/30 border-b">
                Scripts
              </p>
              {useCase.scripts.map((s) => (
                <ScriptRow key={s.file} script={s} />
              ))}
            </div>
          )}
        </div>
      )}
    </div>
  );
}

// ── Main view ─────────────────────────────────────────────────────────────────
export function CapabilityMatrixView() {
  const { matrix, loading, error } = useCapabilityMatrix();
  const navigate = useNavigate();

  function handleLaunch(agent: MatrixAgent) {
    // Pre-fill the Orchestrator textarea with the agent label prefix
    // Navigation passes state so OrchestratorView can seed the textarea
    navigate('/orchestrator', {
      state: { prefill: `${agent.label}: ` },
    });
  }

  if (loading) {
    return (
      <div className="flex items-center justify-center h-full
                      text-muted-foreground text-sm">
        Loading capability matrix…
      </div>
    );
  }

  if (error) {
    return (
      <div className="p-6 text-sm text-red-600">{error}</div>
    );
  }

  if (!matrix) return null;

  return (
    <div className="flex flex-col h-full overflow-hidden">
      {/* Header */}
      <div className="px-6 py-4 border-b shrink-0">
        <h2 className="text-base font-semibold">Capability Matrix</h2>
        {matrix.generated_at && (
          <p className="text-xs text-muted-foreground mt-0.5">
            {matrix.generated_at}
          </p>
        )}
      </div>

      {/* Use case list */}
      <div className="flex-1 overflow-y-auto p-4">
        {matrix.use_cases.map((uc) => (
          <UseCaseSection
            key={uc.title}
            useCase={uc}
            onLaunch={handleLaunch}
          />
        ))}
      </div>
    </div>
  );
}

`useState(prefill)` seeds the textarea on mount. If navigated to directly

## Notes

**Parser robustness.** The matrix Markdown is machine-generated by
`capability_matrix_assemble.exs` — the format is stable. The line-by-line
parser handles the known structure. If the format ever changes, the parser
will return empty use cases rather than crash (the filter step removes them).

**Duplicate agent rows.** The matrix has duplicate entries for some agents
(e.g. `at1cmd.exs` appears three times with different labels in the api/tenant
section). The parser keeps all rows as written — deduplication is not needed
since each row represents a distinct agent role.

**Tool parsing.** Tools are backtick-wrapped and comma-separated in the
matrix: `` `run_command`, `spawn_agent` ``. The parser strips backticks and
splits on comma. MCP server entries like `(MCP servers: corpus_search,
lattice)` will parse oddly — the tools field will contain `(MCP servers:
corpus_search` and `lattice)`. This is acceptable for now; display as-is.

**`useLocation` state type.** Cast as `{ prefill?: string } | null` to avoid
`any`. If the router's `state` type is already typed elsewhere in the codebase,
follow that convention instead.

**`useState(prefill)` vs `useEffect`.** `useState(prefill)` sets the initial
value once on mount. If the user navigates away and back with a different
prefill, React won't re-initialise — but this scenario doesn't arise in the
Option A flow (each launch is a fresh navigation). `useState` is correct here.
